> source attribution, and an evaluation harness that measures whether the answers are actually right.

## What it is

Ask a question, get an answer that **quotes its sources** and says *"I don't know"* when the
retrieved context doesn't support an answer. Runs entirely locally on a single GPU — no API keys,
no per-token cost — using Ollama for both embeddings and generation.

## Architecture

```
Documents → Chunk (overlap + metadata) → Embed (Ollama) → Postgres + pgvector
                                                               │
Query → Embed → Hybrid retrieve (vector + keyword) → Rerank → top-k chunks
                                                               │
                       Gemma (Ollama) + retrieved context → answer + [citations]
                                                               │
                                  eval/ harness → hit-rate@k, faithfulness
```

See [docs/architecture.md](docs/architecture.md) for the detailed flow.

## Stack & rationale

| Layer | Choice | Why |
|------|--------|-----|
| Vector store | **Postgres + pgvector** | Production-grade, SQL-backed; not a throwaway in-memory store |
| Embeddings | **`nomic-embed-text` (Ollama)** | Local, free, runs on one GPU |
| LLM | **Gemma (Ollama)** | Local generation, no API cost |
| Backend | **FastAPI** | Async, typed, streams responses (SSE) |
| Frontend | **React + TypeScript (Vite)** | Streaming chat UI with inline citations |
| Infra | **Docker Compose** (→ k8s manifest) | One command to run the whole system |

## Corpora

The engine is corpus-agnostic via a small [`CorpusLoader`](api/corpora/base.py) interface.
Shipped adapters:

- **`markdown_dir`** — point it at any folder of `.md` / `.txt` files.
- **`crystal_battle`** — a competitive-Pokémon meta analyst over Smogon usage stats, Pokédex
  data, and battle replays from the [crystal-battle](https://github.com/GrumpyRumpus) project.
  Ask *"What's the most common Heatran moveset on Fire teams?"* and get a cited answer from the
  real usage data.

## Run it

```bash
cp .env.example .env            # set CORPUS, OLLAMA_HOST, model names
docker compose up -d            # postgres+pgvector, api, web
python -m api.ingest            # build the index from the configured corpus
# open http://localhost:5173
```
